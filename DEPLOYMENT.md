# Render Deployment Guide

This guide explains how to deploy the Shift Management System to Render.

## Prerequisites

- A Render account (free tier available)
- Git repository with your code
- Gmail account with app password for email notifications

## Backend Deployment

### 1. Push Code to Git

```bash
git add .
git commit -m "Add Render deployment configuration"
git push origin main
```

### 2. Deploy to Render

1. Go to [render.com](https://render.com) and sign in
2. Click "New +" → "Web Service"
3. Connect your Git repository
4. Render will automatically detect the `render.yaml` file
5. Review the configuration:
   - **Name**: shift-management-api
   - **Runtime**: Python 3.11
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn -w 4 -b 0.0.0.0:$PORT app.backend.app:app`

### 3. Configure Environment Variables

In the Render dashboard, set these environment variables:

**Required Variables:**
- `DATABASE_URL` - Auto-linked from PostgreSQL database
- `SECRET_KEY` - Auto-generated by Render
- `JWT_SECRET_KEY` - Auto-generated by Render

**Email Configuration (Required for notifications):**
- `SMTP_HOST` = `smtp.gmail.com`
- `SMTP_PORT` = `587`
- `SMTP_USER` = Your Gmail address
- `SMTP_PASS` = Your Gmail app password (not regular password)

### 4. Create PostgreSQL Database

The `render.yaml` file includes a database configuration. Render will automatically:
- Create a PostgreSQL database named `shift_db`
- Link it to your web service
- Set the `DATABASE_URL` environment variable

### 5. Deploy

Click "Create Web Service". Render will:
- Build your application
- Install dependencies
- Start the server with gunicorn
- Run the email notification scheduler

## Frontend Deployment (Optional)

If you want to deploy the React frontend separately:

1. Create a new Static Site on Render
2. Connect to the same repository
3. Set build command: `cd app/frontend && npm install && npm run build`
4. Set publish directory: `app/frontend/dist`
5. Add environment variable: `VITE_API_URL` = Your backend URL

## Email Setup for Production

To enable email notifications in production:

1. **Generate Gmail App Password:**
   - Go to Google Account settings
   - Enable 2-Factor Authentication
   - Go to Security → App Passwords
   - Create new app password
   - Use this as `SMTP_PASS`

2. **Set Environment Variables in Render:**
   - `SMTP_USER` = `your-email@gmail.com`
   - `SMTP_PASS` = `your-16-char-app-password`

## Verification

After deployment:

1. Check the Render logs for "Email notification scheduler started"
2. Test the API endpoints
3. Verify database connectivity
4. Monitor email sending in logs

## Troubleshooting

**Build fails:**
- Check Python version compatibility (3.11+)
- Verify all dependencies in requirements.txt
- Check Render build logs

**Database connection errors:**
- Ensure PostgreSQL database is created
- Verify DATABASE_URL environment variable
- Check database is linked to web service

**Emails not sending:**
- Verify SMTP credentials are correct
- Use Gmail app password, not regular password
- Check Render logs for email errors

**Scheduler not running:**
- Look for "Email notification scheduler started" in logs
- Ensure APScheduler is in requirements.txt
- Check for Python errors in startup

## Scaling

For production use:

- Increase gunicorn workers: `-w 8` instead of `-w 4`
- Add Redis for session storage
- Use Render's paid tier for better performance
- Set up monitoring and alerts
